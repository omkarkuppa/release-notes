  [TSE] Fixed BlockIoCrypto protocol to correctly report AlignIo
  
  [Issue Description]
  The BlockIoCrypto protocol was incorrectly reporting the Media.AlignIo
  value instead of the expected CryptoMedia.AlignIo value.
  
  [Resolution]
  The BlockIoCrypto protocol now properly reports the CryptoMedia data
  structure, which is initialized with the same values as Media but with
  an updated AlignIo field reflecting the 16B alignment requirement from
  TSE.
  
  To ensure validity, a check is performed to verify consistency between
  the Media and TSE's CryptoMedia alignment requirements: if 16 is not a
  multiple of Media.IoAlign, TSE is disabled for the corresponding block
  device.
  
  Refactored code to encapsulate TSE requirements checks in a standalone
  function.

  TseNvmExpressDxe
