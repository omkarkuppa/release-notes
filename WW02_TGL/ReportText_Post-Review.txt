[SiliconPkg], [BoardPkg]

  Description:[TGL][MRC] LP5 enablement with ECT
  
  Solution:
  update mLpddr5xQdp12Gb496bSpd in PeiSpdTables.c
  add support for LPDDR5 in GetSpdData (MrcOemPlatform.c)
  Add suport for the following frequencies in CMOS override 0x4B:
  2200, 2800, 3600, 4000, and 4800
  added ucode 0x48 for SAGV stability
  
  LP5 ECT enabling
  1) Frequency switch at the begining of ECT was forcing Gear1 but LP5 POR is Gear2. So frequency switch should happen selectively for lp4 and lp5.
  2) For Lp5, CADB output are not synchronized to CkSyncCounter in MC. In order synchronize the CADB output we need to delay the CADB start. There is a counter in the CADB which decrements based on this StartDelay value, but the counter ignores any value < 4. Therefore CADB Start Delay should be set to 4 for all channels. This was found in preSi testing but wasnt being correctly set for ch2, ch3, ch6 and ch7.
  3) RTL workaround for LP5 was removed for B0 but when it was removed the function call MrcDriveDq7 was also removed when we enter CBT mode. Therefore Dq7 was never asserted high, DRAM didnt switch the FSP thereby never sending any feedback on the dq pins.
  4) Changed the starting values of CMD, CTL and CLK Pi to the ones recommended by the PHY team.
  5) Commented out the Cbtphase1. For falling edge CMD sampling MRC was programming the CADB unisequencers with the same pattern as rising edge shifted by one cycle of 0 pattern. But on scope we found that the pattern was shifted by 1 tCK instead of half tCK. Until we program the CADB BUF and Unisequencers correctly, cbtphase1 training is disabled.
