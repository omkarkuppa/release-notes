  C1S/IBL/GPIO: Handle virtual GPIO Pad Config registers- resubmit
  
  Originally the change has ben reverted because of post-merge test fail:
  EagleStreamRp_SPR_EBG_FSP_Glue_Debug
  Now the change is resubmited.
  
  For virtual GPIO we have different configuration registers layout.
  To support both GPIO types (physical and virtual) in the same library
  we have to handle two types of register layouts.
  This change introduces virtual GPIO register layout support in
  current GPIO libraries.
  
  Following settings are not supported for virtual GPIOs:
  - GPIO input inversion (RXINV)
  - GPIO RX raw override to '1' (RXRAW1)
  - GPIO Pad Mode (PMode)
  - GPIO termination (Term)
  Additional settings for virtual GPIOs:
  - Pad Ownership (PADOWN)
  - HOSTSW_MODE
  - vGPIO_OUT_CS
  
  Additional changes in Pad Configuration Register:
  - TX State moved to bit1
  - RX State moved to bit2

  The update has been based on version V.29 of GPIO HAS

  we have to handle two types of register layouts.

  current GPIO libraries.

  - Pad Ownership (PADOWN)
  - HOSTSW_MODE
  - vGPIO_OUT_CS
