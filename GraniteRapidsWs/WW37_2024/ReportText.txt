  [GNR] DRAM CA Vref RMT shall add MR11 as 2nd condition for Pass/Fail criteria
  
  [Issue Description]
  The MPC to set DRAM CaVref may silently fail during a margin sweep. MRC assumes Vref has been updated, but actually the DRAM is in a failed state and the SweepLib should treat the current CaVref setting as a failure.
                                 ^^^^^^                                                                                                                                                                                    
  
  [Resolution]
  The solution is for MRC to perform an MRR of MR11, after the MPC, to confirm if the MPC was successful. If the MRR results doesn't match what MRC things it wrote with the MPC, then the Vref setting is failing.
  
  Besides, MRR command failure is detected during DRAM CA VREF RMT, the trace shows the returned code from MR11 via MRR command, it can be either "valid data on upper or lower with random value" or "invalid data in entire byte".
  Current ReadCpgcMrrData would treat invalid data as error and system would hang, however, it can be just command failure since DRAM CA Vref is swept to boundary of margin. To solve this, MRC shall add command test mode support
  for ReadDramMr API to detect bad command being send during DRAM CA Vref sweep.

[CPU]

  Enable X2APIC for BSP in pre mem phase for specific processor
  
  [Feature Description]
  In MpInitLibInitialize() of PeiMpLib:
  1.MpInitLibInitialize saves the BSP's initial APIC mode and
  syncs to all APs in the first time wakeup in CollectProcessorCount().
  2.If the BSP's Apic mode is LOCAL_APIC_MODE_XAPIC, AutoEnableX2Apic()
  will forcely enable x2APIC mode for both BSP and AP if needed.
  
  Reccently we found that the calling of WakeUpAP in step2 cost a lot of
                                                     ^^^^                
  time on a specific processor. To reduce the time cost of MpInitLibInitialize,
  we can enable X2APIC for BSP in pre mem phase. After that, AP's APIC mode
  will be synced to X2Apic mode during the first call of MpInitLibInitialize
  in CpuMpPei. Then the step2 in MpInitLibInitialize can be skipped.
                        ^^^^
