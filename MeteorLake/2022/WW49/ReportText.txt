  [TSE] BlockIoCrypto support for unencrypted IO & custom tweak values.
  
  [Feature Description]
  
  The BlockIoCrypto protocol API defined in the UEFI specification
  describes two parameters for read & write extended operations:
  Index and CryptoIvInput:
  
  1) BIOS will consume the optional Index parameter to determine if
  the operation should be handled by the BlockIoCrypto protocol
  performing inline encryption through the TSE engine or deferred
  to the BlockIo protocol to perform a regular NVMe IO.
  
  2) BIOS should consume the caller's CryptoIvInput as a
  EFI_BLOCK_IO_CRYPTO_IV_INPUT_AES_XTS structure as defined in
  the UEFI spec to determine the tweak value for the AES XTS
  algorithm. This parameter becomes mandatory for the
  BlockIoCrypto protocol supporting TSE.
