  [MTL-P][CRB] Enable S0ix Support for MTL-P CRB Platform
  
  [Issue Description]
    MTL-P CRB system not able to get s0ix value when enter Modern Standby mode
    due to shared CLKREQ#/REFCLK across multiple root ports
  
  [Resolution]
    For S0ix implementation an independent CLKREQ#/REFCLK signal is required per root port and
    modifying the existing sharedCLK configuration will block the usage of other device on the root port.
    so for S0ix support, new FAB-ID is created with which S0ix can be validated.
