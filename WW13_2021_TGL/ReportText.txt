[SiliconPkg]

  [C1S/MemoryInit/TGL]: Fix the Fast Boot failure when ECC DIMMs are used
  
  [Issue]
  With ECC Dimms, MarginLimitCheck failed during Fast Boot.
  The GsmMccEccMode was only set on MrcMcActive() which was not executed during Fast Boot.
  
  [Solution]
  Program the GsmMccEccMode to emBothActive in the MrcNormalMode(), which covers all PM flows.
  During saves/restores, GsmMccEccMode are set to emEccIoActive instead of emBothActive for training steps.
  
  Additional Changes:
  Add ECC byte error check support on MrcGetMiscErrStatus.
  Improve debug message in MarginLimitCheck.
  Use McChBitMask instead of ValidChBitMask in MarginLimitCheck. (since they are the same)
  New EFI status code: EFI_COMPUTING_UNIT_MEMORY | EFI_CU_MEMORY_EC_CORRECTABLE when MarginLimitCheckFailed.
  
  [Impact Platform]
  TGL-H ECC DDR4 Only
