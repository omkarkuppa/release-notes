  [HSD-ES][client_platf][bug][14013058363][ADL-S][DDR5][Early Command Training fails with DDR5 ECC modules]
  
  Issue:
  Early Command Training fails with DDR5 ECC modules
  
  Root cause:
  ECC byte has only 4 bits in DDR5, but DataTrainFeedback mask is 0xFF for all bytes in CSTM/CATM
  
  Solution:
  Use 4-bit mask for ECC byte in CSTM/CATM
  
  Additional fixes:
  - ProgramDeswizzleRegisters: skip ECC byte in MC DESWIZZLE programming
  - DQ mapping: Use 4-bit mask for ECC byte feedback and only map bits [0..3]
  - Various training steps: MRC_DDR5_ECC_BYTE should be 4 instead of 5
  - JWL: use proper 4 bits in ReadAfeMiscControl0Reg() to get correct feedback on the ECC byte
  - DQTimeCentering1D: use 32 + 4 = 36 bits in DQ results print (instead of 40)

  [C1S/AMT] AMT Configuration Menu in BIOS Setup is accessible when AMT KVM session opened while SUT in BIOS
  Merge from TGL.
  
  Issue: Remote KVM session is opened after BIOS sends Get ASF Boot Option command, therefore the KVM in Boot Option is 0
  Resolution: Use USB-R port to check if KVM is enabled in runtime, and modify the flow as:
  1. If remote KVM is opened before entering or exiting "PCH-FW Configuration", BIOS can detect the KVM and update the accessibility accordingly.
  2. If remote KVM is opened in "PCH-FW Configuration", because BIOS can't update the accessibility, BIOS will pop out a dialog box and reset system.
  Impacted: All C1S projects.
